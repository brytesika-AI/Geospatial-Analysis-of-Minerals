Technical Architecture · Data Science · AI

System Architecture

End-to-end design covering data engineering, machine learning, mineral systems modelling, AI interpretation, and cloud deployment — built for KoBold Metals-style exploration operations.

Business Context

GeoExplorer AI is a mineral prospectivity screening platform targeting Cu, Co, Ni and green critical metals across Sub-Saharan Africa. The platform addresses a specific exploration challenge: reducing the cost of identifying drill-ready targets in the Sediment-Hosted Stratiform Copper (SHSC) deposit systems of the Lufilian Arc (Zambia Copperbelt, DRC Katanga), Bushveld Complex (South Africa), and satellite green-metals districts across the continent.

Strategic Value

  • Triage: 20,125 grid cells ranked by ML prospectivity → focus field time on top 1% of the search space
  • Diagnostics: Co/Cu ratio flags DRC Katanga-type SHSC; Ni/Cu flags Bushveld magmatic sulphide
  • Green metals: Li (Manono, Ewoyaa), Mn (Kalahari), REE (Ngualla, Mrima Hill) — energy transition overlay
  • Logistics: Infrastructure layer (518 cities, 68 ports, 593 railway segments) informs capex and concentrate routing
  • Decision output: Field-ready GPS coordinates, drill method, budget, permit authority per jurisdiction

Data Sources

  • USGS MRDS WFS (WFS 1.0.0): Global mineral deposit registry — Cu, Co, Ni, Li, Mn, REE, Graphite. Fixed bbox axis-order bug (lon/lat for WFS 1.0.0, lat/lon for 1.1.0 EPSG:4326).
  • GEM Global Active Faults: ~40MB worldwide fault network clipped to Africa — 323 segments used as structural proximity features.
  • OpenStreetMap Overpass API: Real mine/quarry locations within the Africa bbox — live data refresh on pipeline runs.
  • Natural Earth Vector: Populated places (518 SSA cities pop>50k), seaports (68), country polygons (52), railways (593 segments).
  • Geochemistry: Calibrated synthetic dataset (no free Africa soil geochem API exists) based on Tembo et al. (2009), Cailteux et al. (2005), USGS PP 1802. Log-normal distributions calibrated to known Copperbelt anomaly thresholds.
  • SRTM Elevation: 90m DEM via terrain proxy (srtm.py) — slope and elevation features.

Machine Learning Architecture

Feature Engineering (13 features)

  • log(Cu ppm) — primary pathfinder
  • log(Co ppm) — DRC Katanga diagnostic
  • log(Ni ppm) — Bushveld diagnostic
  • log(Au ppb), log(Pb ppm)

  • dist_deposit_km — district proximity

Model Selection & Validation

  • Algorithm: XGBoost (gradient boosted trees) selected via 5-fold spatial cross-validation against Random Forest and Logistic Regression
  • Spatial CV: Geographically blocked folds prevent data leakage from spatial autocorrelation — critical for geological AI models
  • ROC-AUC: 0.889 · PR-AUC: 0.785 — strong performance given synthetic geochem labels
  • Scoring: 20,125 grid cells at 0.1° resolution (≈11 km) across Central/Southern Africa (15°E–38°E, 35°S–0°)
  • Point scoring: IDW interpolation from 4 nearest grid cells — serves the /api/score endpoint in <50ms

Mineral Systems Semantic Model (SHSC Framework)

Each ML feature is mapped to a component of the SOURCE → PATHWAY → TRAP → MODIFIER framework (Hitzman et al. 2010). This grounds model outputs in geological theory and drives the LLM geological interpretation layer, ensuring AI outputs are geologically coherent rather than purely statistical.

AI & LLM Interpretation Layer

  • Model: Meta-Llama-3-8B-Instruct via HuggingFace Inference API
  • Prompt engineering: Structured system prompt with 800-word African Copperbelt geology context (Lufilian Arc, Katanga Supergroup, Bushveld BIC, SEDEX Northern Cape)
  • Output format: 4–6 bullet geological assessment: score interpretation → SHSC mineral system evidence → limiting factors → recommended next step
  • Fallback: Deterministic rule-based interpretation when HuggingFace API is unavailable — based on Co/Cu, Ni/Cu, structural proximity thresholds
  • Co/Cu diagnostic: >10% → DRC Katanga; Ni/Cu >20% → Bushveld magmatic sulphide; else Zambia Copperbelt SHSC or Namaqualand

Key Technical Decisions

  • Pre-computed grid vs. live inference: 20,125 scores pre-computed offline; API serves CSV lookup + IDW — eliminates cold-start latency and removes XGBoost from serverless runtime
  • No system GDAL dependency: GeoJSON loaded with Python stdlib json module — enables deployment to any Python serverless environment without binary dependencies
  • Spatial CV over random CV: Prevents optimistic AUC inflation from spatial autocorrelation — critical methodological choice for geological ML
  • Log-normal geochem normalization: Cu, Co, Ni, Au, Pb, Zn, Mo, As all log-transformed before model ingestion — correct for heavy-tailed geochemical distributions

Key Insights

  • Dist. to deposit is the single highest-importance feature — district-scale proximity overwhelms individual geochemical signals, consistent with SHSC deposit clustering
  • Co signal ranks above Cu in importance — reflects the DRC Katanga Co-dominant systems in the training data; Co anomalies are rarer and more diagnostic
  • Fault proximity (dist_fault_km) ranks 3rd — validates the structural control hypothesis for SHSC fluid focussing in the Lufilian Arc
  • Mo inversely associated with prospectivity — consistent with SHSC vs. porphyry discrimination; high Mo indicates a different mineral system
  • Highest-scoring cells cluster in the Zambia Copperbelt extension (Chililabombwe-Ndola trend) and northern DRC Katanga (Kolwezi-Tenke corridor), validating the model's geological coherence
  • South Africa model extension: Bushveld BIC targets identifiable by Ni-dominant signal; Kalahari Copper Belt shows SHSC-type Co/Cu pattern consistent with the Zambian system
